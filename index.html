Organizing Tests

  • tests for functions belong to "ext/standard/tests"
  • extensions tests are in "ext/standard/<extension>/tests"

Naming conventions

  • <functionname>_basic.phpt
  • <functionname>_variations.phpt
  • <functionname>_error.phpt
  • bug<bugid>.phpt
  • <extname><no>.phpt

Number suffix for duplications

Structure of a test


--TEST--
strtr() function - basic test for strtr()
--FILE--
<?php
/* Do not change this test it is a README.TESTING example. */
$trans = ["world" => "PHPUGMS"];
var_dump(strtr("# Hello, world! #", $trans));
?>
--EXPECT--
string(19) "# Hello, PHPUGMS! #"

Available sections

  • --TEST--
  • --DESCRIPTION--
  • --SKIPIF--
  • --HEADERS--
  • --FILE--
  • --EXPECT--
  • --CLEAN--

All sections with proper description: PHPT - Test file layout

  • Test: 1 Zeile
  • Desc: Länge Beschreibung
  • Skip: OS, 32/64Bit, CLI, ...
  • Headers: Http Request Headers
  • File: Der Code
  • Expected: Das erwartet Ergebnis
  • Clan: Aufräumen

--SKIPIF-- Example


--SKIPIF--
<?php
if (PHP_INT_SIZE != 8) {

?>

--TEST--
